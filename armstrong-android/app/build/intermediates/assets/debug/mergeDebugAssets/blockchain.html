✅ Why Armstrong Uses Firebase Realtime Database

For a real-time maternal health monitoring system, speed, security, and open-source reliability are critical. Armstrong uses Firebase RTDB as its primary backend — it provides sub-100ms push updates via WebSocket when the TI-Band sensor hardware writes new vitals, while keeping all data in a structured JSON tree.

Blockchain was evaluated but rejected for primary storage because: gas fees per vital sign update would be cost-prohibitive (vitals update every 2–3 seconds = ~40,000+ writes/day), and block confirmation latency is unacceptable for emergency alerts.

Firebase Security Rules ensure only authenticated devices can write, and only the patient's doctor can read their data — achieving HIPAA-grade access control without blockchain complexity. The Firebase sensor_data/device_001 reference is the single source of truth for all vitals.
